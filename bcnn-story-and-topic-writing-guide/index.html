STEP 1: WRITE THE STORY
Before you can upload a story, you have to write the story. However while we have no desire to tell you how to do your job to complete excellence, there are somethings to consider before you start uploading your work.
Story: Topics covered
What topics do your story cover? Often, a story may cover a single or multiple topics. While this shouldn't affect the writing process, it's something to keep in mind for later.
Title
Of course, a story or a topic page needs a title.
Story: Author
All reporters are allowed to write under any name they want. They can change this name between stories if they so wish.
Story: Date
When was this story published and uploaded? When do you want it to be published?
Sub-header
Sub-header can be used to section off the story into chunks. If you choose to use these, then this BCNN system has support for them.
There are some other things to consider as well.
Images
This system does have support for images, however the process for adding images into a story is slightly more complicated. We will cover it later on in this guide.
Bold and Italic
This system offers support for bolding and italicizing text, however it's a bit more complicated than just pasting bold or italic text into the Google Sheets document. If you want to bold or italicize anything, use the following format:
<strong>Text here is bolded</strong>
<em>Text here is italicized</em>
So you must surround your bolded or italicized text with the correct tags. So for example, if I want to write the following sentence:
While he asked for a large volume of University of Stone copies, Phillip Ball demanded he would have one large copy of University of Stone, volume one.
Then I would need to write the following in my story:
While he asked for a <em>large volume</em> of University of Stone copies, Phillip Ball demanded he would have <strong>one large copy of University of Stone</strong>, volume one.
Additional tags can be found at this link: https://www.geeksforgeeks.org/html-tags-a-to-z-list/. It should be noted that any HTML text interset into the story will be treated as HTML when processed into the story. This can potentially be used for creating many things, including bulleted lists, tables, and more.
Links
Links are a great way to immediately connect readers to important pieces of information related to the story. This is done through the use of tags. If you want to include a link, use the following format in your story:
<a href='[INSERT LINK TO PAGE HERE]'>[TEXT WITH LINK HERE]</a>
In the [INSERT LINK TO PAGE HERE] section, paste in the link that the reader will be sent to when the link is clicked. You can put anything in the [TEXT WITH LINK HERE] section; it will become blue/purple and will send users to the link when clicked.
As an example, let's say I wanted to write the following text:
Phillip Ball has established de facto control over all of Earth-4.
Then I would need to write the following text into my story:
<a href='https://bcnnofficial.github.io/topics/phillip-ball/'>Phillip Ball</a> has established de facto control over all of <a href='https://bcnnofficial.github.io/topics/earth-4/'>Earth-4</a>.
Footnotes
This system also has support for footnotes. To add a footnote into a story, paste the following code straight into your story immediately after the word you want to be footnoted.
<sup style="font-size:70%"><span class='footnote' id='[ID HERE]'onclick="FootNote('[ID HERE]','[TEXT IN FOOTNOTE HERE]');">[a]</span></sup>
This allows you to add footnotes anywhere in your story, like so[a].
Some things to note are the [ID HERE] and [TEXT IN FOOTNOTE HERE] sections. The latter is pretty straightforward; it is where you put the text you want to pop up when the footnote is clicked. If you want apostrophes in the footnote text, type " \' " rather than " ' " (without the quotes; must be a backslash, not /). Currently, this system does not allow for quotation marks in the footnote text; we apologize for this inconvinience. The [ID HERE] section is a unique identifier that the footnote needs to work. It must be a word without spaces, quotation marks, or apostrophes, and each footnote needs a different id.
As an example, if I wanted to write the following text:
Phillip Murderer[b], wanted for the murder of Phillip Victim[c], has finally been caught and executed on the spot.
Then I would need to write the following in my story:
Phillip Murderer<sup style="font-size:70%"><span class='footnote' id='ftnB'onclick="FootNote('ftnB','Not to be confused with Phillip Murder.');">[b]</span></sup>, wanted for the murder of Phillip Victim<sup style="font-size:70%"><span class='footnote' id='ftnC'onclick="FootNote('ftnC','Not to be confused with Phillip Murdered, who was killed by Phillip Murder.');">[c]</span></sup>, has finally been caught and executed on the spot.
Of course if you are having difficulty adding footnotes or anything into your story, then feel free to reach out to our Writing Assistnace Office.
STEP 2: EDIT STORY INTO GOOGLE SHEETS DOCUMENT
For our system to work, it needs a little bit of help. Below are instructions for how to parse your story into the Google Sheets document so it can be converted into a site correctly.
The systems are slightly different depending on if you're making a story or a topic page, so follow the instructions as necessary.
FOR A STORY
To upload a story, it must be inserted into this Google Sheets document.
If you are creating a new story, then you must make a new sheet: click on the plus icon in the bottom right corner.
You can name the page whatever you want (don't use quotation marks). You'll need the story sheet name later on.
Now you'll want to set up your sheet as follows:
For the id row, you will start at 1 and count up as you go on.
TOPICS
First, you'll need to specify which topics, if any, your story is connected to.
Under info1, type in "TOPIC" (must be all caps). Under info2, type in the name of the topic. Under info3, insert the link to the topic page if one exists for it.
You can have as many, or as little, topics in your story as you want. However it's advisable to insert them before anything else.
TITLE
This is where you insert the title of your story.
Under info1, type in "TITLE". Under info2, add your title. Most titles are in all caps, but this is not required.
Note that until we start adding paragraphs, the order in which these are added matters. The website may not be created in the right order if things are not added in the demonstrated order.
AUTHOR, DATE
Stories commonly contain a listed author and the date they were published.
Under info1, enter "AUTHOR" and "TITLE". Under info2, enter the author of the story and the time and date of publication.
IMAGE
At this point, imgages, subheaders, and paragraphs can be added in any order. However it may be preferrable to start off your story with an image.
To add an image, under info1, type "IMAGE".
Under info2, type in the file name for the image. For example, in this case the file name is "phillip wisdom.jpg".
Under info3, type in the alt text for the image. This is just the text that appears when the image doesn't load correctly.
Under info4, type in the caption for the image. This text will appear under the image on the story page.
Under info5, add any additional information about the image; a footnote will be created which will display this information when clicked (you don't need to add the footnote yourself). Add information here about image sources, if the image is AI generated, etc. Make sure you are licensed to use anything that appears in the image, and give attribution when necessary. Some media may have a creative commons license, but may legally require attribution or may not allow for derivative works.
If you have nothing you'd like to be in info5, type "NONE" in all caps.
PARAGRAPHS
Each paragraph of your story must be added separately.
Under info1, type "PG". Under info2, paste the contents directly into the cell. Keep in mind that any formatting you do yourself will be ignored, and only the formatting that the system will recognize is detailed above in Part 1.
SUBHEADERS
Subheaders refer to smaller titles that can section a story into pieces.
Under info1, type "ST". Under info2, paste in the text of the subheader.
COMMENT SECTION
Many BCNN stories include a sort-of comment section that can be accessed at the bottom of the page.
Under info1, type "COMSEC". Under info2, paste the link to the comment section.
Now that's all grand, but how to we create a comment section for this story?
First, make sure you are logged into the official BCNN Google account. If you don't have access to this account, contact the Writing Assistnace Office.
Then, find "Blogger" in the collection of apps.
Then, create a new blog post.
Design the post however you want; what's important is that when the post is published, a comment section will appear at the bottom of the page. Click publish when done.
To go to the blog page, click on the view icon.
Then finally, copy the link.
Honestly this system is pretty horrible so if you have any better ideas, go for it. You just need to post a link to it under info2. But you can use anything at all which would act as a comment section. Doesn't have to be connected to Google or anything.

STEP 3: SET UP STORY FOR GITHUB
Though not the last step entirely, this is the last step until your story is created and live on BCNN. The process for adding a story versus adding a topic page is slightly different, so follow directions as necessary.
FOR A STORY
You will need access to the BCNN github page for the following. If you do not have access, please contact the Writing Assistance Office.
Within the "stories" folder, there should be another folder called "template", with a file in it called "index.html". Open that file and copy all of the text inside of it.
Next, go back to the stories folder. While inside of this folder, click on "Add file" in the top-right corner, then click on "Create new file".
At the top, you will be asked to add a name for the file. Your file name will be "[name of folder]/index.html".
In this case, I typed in "sample/index.html". This makes a folder inside of the "directories" folder, and creates the index.html file inside of that folder. You can name the folder whatever you want, but keep in mind that whatever you name that folder, it will become part of the page URL. For example, this story will have the link "https://bcnnofficial.github.io/stories/sample/". You create the folder when you type in the forward slash. Keep in mind the file name must be index.html, or the site won't work.
Next, paste in all the code from the template.
When you first paste it in, under <script>, it should read GenerateStory("[ENTER SHEET STORY NAME HERE]", "18UsYDlaHZrCuIUt9FOMiU6zP7se5EVXjsHgttUDXzIs");. In that [ENTER SHEET STORY NAME HERE] area, paste in the name you gave your sheet in the Google Sheets document. After that, you can hit "Commit changes".
Finally, any images you may have used in your story, you will need to upload them into the folder.
In the folder that has your index.html file, click on "Add file", then "Upload file". Choose the image you need to upload. After that, you should be done. Your site can be found at "https://bcnnofficial.github.io/stories/[file name]/".

If at this point, you are able to access the page, and everything looks good to you, we can move to the final step to fully integrate your story or topic into the BCNN mainframe framework work system.
STEP 4: ADD INFO TO OTHER GOOGLE SHEETS
While everything may be done with your story, there are some last things to do. Once a story is fully integrated, it can appear on the topic pages it's associated with. It can appear at the bottom of the page as a suggested story. And it can appear other places.
What needs to be done depends on whether you have written a story or a topic page. Follow the directions as necessary.
But before we begin, you'll be asked in both sections to provide image links. The process for this is slightly different than before. Since the image won't always be in the same folder compared to where it's being used, you will need to grab a universal link.
First, find the image in your folder containing the index.html file. Click on it to open the image in github.
Right-click on the image, and select the "Copy image address" option. You now have a link to the image that can be used from anywhere on the page.
FOR A STORY
You will have to complete the following process on multiple sheets. To access these sheets, click here.
First, you'll have to add your story to the "All" sheet no matter what. This is where all stories are kept. You will also have to add your story to any topics it is part of, as well as to any "story collections" (the sheets with the really long names). If you want to make your own story collection, that process will be demonstrated at the end of this section.
Regardless of what sheet you have access to, it should look like something from the image below:
Similar to before, if you are adding a new story, you'll have to increment the number under "id".
Under "name", paste in the title of your story.
Under "img_link", paste the universal link to the image you want to appear in thumbnails for the story.
Under "date", paste in the date of publication.
Under "first_paragraph", paste in the entire first paragraph from your story.
Under "link", paste in the link to your story.
Under "disclaimer", paste in any disclaimer associated with the image, like if it is AI generated or whatever.
Under "alt", paste in the alt description for the image.
You do not need to worry about any of the other columns. If everything goes according to plan, your story should now be findable in all the right places.
The below content is if you want to make your own story collection. "Story collections" are collections of stories that have a similar subject, and they are found on the "Need to Know" page (link). If you are making stories that are all about a similar subject and you want to organize all your stories into one place, then this might be helpful.
To add a story collection, you need to access this Google Sheets document.
Add the next highest number under "id".
Under "name", add the name you want to appear for your story collection.
Under "description", provide a brief description of what your story collection covers.
Under "color", provide the color of the box you want all the info to appear in. There is a provided link that has a list of color accepted by the system. You can also provide a hexidecial value in the form of #xxxxxx. You can seach "color picker" on Google and it will provide you with a tool that lets you pick a color and find its hex value.
Under "topic", give some keyword or keywords connected by hyphens that relate to your story collection.
Now, go back to the story Google Sheets page (link), and add a new sheet by clicking the plus icon in the bottom right. Let the name of this sheet be the name of the story collection.
Design the first row to match how it does below:
Have the first row contain "name", "img_link", "date", "first_paragraph", "link", "disclaimer", "alt", and "topic". Under "topic", type in the keyword or keywords you came up with previously in the story collection sheet.
At this point, your story collection should appear as it's supposed to in the Need to Know page. You can add stories into it now.
FOR A TOPIC PAGE
First, you will need to access the topics Google Sheets page (link).
Under "id", add the next highest number.
Under "name", add the name of the topic (customarially not in all caps, however this does not have to be the case).
Under "img_link", add the universal link to the image you want associated with your topic.
Under "link", add the link to the topic page.
Under "disclaimer", add any disclaimer associated with the image, like if it is AI generated or whatever.
Under "first_paragraph", paste in the whole first paragraph from your topic page.
Under "color", add the color of box you want your material to appear in when searched for on the topic searching page. This link provides a list of words you can enter that are associated with colors, however you can also enter a hexidecial value for your color in the format #xxxxxx. This link will provide you with a color picker that can give you the hex value of whatever color.
Under "finder", enter some keyword or keywords connected with a hyphen you want associated with the topic.
The last step is to create a new sheet on the stories Google Sheets page (link).
Click the plus sign in the bottom left to create a new sheet. Make sure this name matches whatever you chose for the name on the topics Google Sheets document.
Make sure your top row matches what is above, with "id", "name", "img_link", "date", "first_paragraph", "link", "disclaimer", "alt", "topic", and "finder".
Under "topic", paste in the name of the sheet, which should be the name chosen for the topic on the topics Google Sheets page. For "finder", paste in whatever you chose for "finder" on the topics Google Sheets page.
YOU ARE DONE
